# Example: acme-ui

A minimal example showing the full pipeline output for a fictional design system called "acme-ui" with 3 components (Button, Card, Dialog).

This example demonstrates:
- **Stage 1 output** — `context/acme-ui/01-decisions.md`
- **Stage 2 output** — `context/acme-ui/02-verified-facts/` (per-component facts + summaries)
- **Stage 3 output** — `context/acme-ui/03-closed-prd.md` (abbreviated)
- **Stage 4 output** — `skills/acme-ui/` (SKILL.md, index, guides, per-component api + examples)

The content is handcrafted to show the format each stage produces. In a real pipeline run, all of this is generated by agents reading actual source code.

## acme-ui (fictional)

- **Package:** `@acme/ui`
- **Version:** 1.0.0
- **Components:** Button, Card, Dialog
- **Styling:** Tailwind CSS with `@theme inline` token mapping
- **Compound pattern:** Separate named exports (DialogHeader, DialogBody, DialogActions)
- **Framework:** React 18 + Next.js 14 (App Router)
